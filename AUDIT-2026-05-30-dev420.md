# teru / teruwm — Full Audit on Zig 0.17.0-dev.420 (2026-05-30)

Triggered by a toolchain bump to `zig-master-bin 0.17.0-dev.420+8086ae176`.
Full line-by-line audit of **101 `.zig` files + the wlroots C glue + build.zig**
(27 module units, adversarially cross-checked). Version `0.6.11`.

## ⚠ Headline — the build break was upstream Zig, not teru

`zig-master-bin 0.17.0-dev.420` ships a **stdlib typo** that breaks the build of
*any* program touching termios on x86_64 — teru is a terminal emulator, so it is
hit head-on:

```
/usr/lib/zig/std/os/linux.zig:8552
  } else if (arch_bits == .alpha) enum(u32) {     ← arch_bits is a TYPE (per-arch import)
  } else if (native_arch == .alpha) enum(u32) {   ← correct (the idiom used elsewhere: lines 2047, 5229, 5819)
```

`arch_bits` (line 33) is `switch (native_arch) { … }` resolving to an imported
module — a *type*. On x86_64 the comptime chain for `pub const V` (termios
control-char indices) type-checks `arch_bits == .alpha` →
`error: incompatible types: 'type' and '@EnumLiteral()'`.

**teru's own source needed zero changes for dev.420.** Worked around with a
patched local lib dir (`~/.cache/teru-zig-lib-dev420-patched` — symlinks all of
`/usr/lib/zig` except a one-line-patched 17 MB `std` copy; no system files
touched). Permanent options: wait for the next `zig-master-bin` (upstream surely
fixes fast), one-line-patch `/usr/lib/zig/std/os/linux.zig:8552`, or pin to the
prior master.

## Medium findings (46) — themes

- **Correctness on untrusted/edge input**: `extractJsonObject` doesn't skip string contents; `teru_send_input` truncates >4 KB; `WmConfig.applyThreshold` rejects `perf_us > 65535`; X10 mouse-release byte wrong; Win32 dead-key double-`ToUnicode`.
- **VtParser gaps**: CSI intermediate dispatch dead (DECRQM `$p` responder unreachable, DECSTR `!p` dropped); **DECAWM (`ESC[?7l`) tracked but never consulted** — text always wraps; OSC-8 hyperlink only attaches on the ASCII fast-path (`Grid.write` omits `hyperlink_id`).
- **Resource lifecycle**: `ProcessGraph` nodes never `remove`d (unbounded map growth); macOS kqueue watcher leaks the watched fd; XWayland scene_tree leaked on unmap→remap; WinPty double-`CloseHandle` on error.
- **Renderer**: cursor drawn unconditionally on the teruwm pane path (DECTCEM `ESC[?25l` ignored, unfocused panes show a solid cursor); SIMD blend uses vector integer divide-by-255 (no HW vector idiv on x86 — defeats the SIMD path); bar `measureWidgets` vs `renderWidgets` width drift; `populateBarDataCache` writes fields no renderer reads.
- **Compositor**: live `set_config border_width`/threshold changes don't re-arrange; touchpad scroll-stop (`delta==0`) spuriously scrolls 3 lines; full XKB keymap serialized to a heap string on **every** modifier event; `repaintBorderOnly` sends full-buffer damage.
- **Hot-reload**: silently drops all string config fields + keybinds on reload.

## Verdict

The toolchain bump exposed an **upstream Zig regression**, not a teru bug — teru
builds and passes on dev.420 unchanged. The audit went deeper than the v0.6.11
pass: it found a **default-build heap overflow** (FontAtlas variant), a
**security exfil bypass** (in-band JSON-key injection), a **UAF** (graph string
ownership), **two OOB** scratchpad paths, and the **standalone-teru CPU-spin**
that the v0.6.11 audit fixed only for the compositor. **All 6 criticals + all 22
highs + 2 safety mediums are fixed and verified green** (build + 537 tests + headless
E2E); the 44 remaining mediums and 60 lows are documented with precise fixes.
