Why teru?

AI Agent Orchestration

When Claude Code spawns agent teams, teru manages panes natively. Process graph tracks parent-child relationships. Pane borders color-code by status. 19 MCP tools for cross-agent coordination.

Built-in Multiplexer

8 tiling layouts (master-stack, grid, spiral, accordion...), 9 workspaces, session persistence. Replaces tmux entirely. One config file instead of three.

No GPU, Faster

CPU SIMD rendering via @Vector. GPU terminals idle 99.9% on text -- uploading textures for two triangles. SIMD blitting is faster and works over SSH, VMs, containers.

1.4MB Total

Single binary. No FreeType, no fontconfig, no OpenGL, no EGL, no GTK. Compare: Ghostty 30MB, WezTerm 25MB, Zellij 12MB, Alacritty 6MB + tmux 4MB.

Features

CPU SIMD Rendering

@Vector alpha blending, no GPU. Sub-millisecond frame times for 200x50 grids.

8 Tiling Layouts

Master-stack, grid, monocle, dishes, spiral, three-col, columns, accordion. Per-workspace layout lists.

16 MCP Tools

Agent-to-agent pane control. Read output, send input, manage layouts, query process graph.

Session Persistence

Daemon mode with detach/attach. Sessions survive terminal close. .tsess format with restart policies.

Configurable Keybindings

Per-mode tables (normal, prefix, scroll, search). Alt-primary shortcuts. Namespaced actions.

Cross-Platform

Linux (X11+Wayland), macOS (AppKit), Windows (Win32+ConPTY). Single codebase.

Vi/Copy Mode

Vim-like navigation, visual selection, word/line motions, yank to clipboard.

Process Graph

DAG of all processes and agents. Parent-child tracking. Status-colored borders.

Scrollback Compression

Keyframe+delta stream compression. 20-50x smaller than traditional ring buffers.
